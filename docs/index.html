Executive finding

Production profile validated

The model card's vLLM 0.24 + two-token MTP path is compatible on GB10 when KV cache is FP8. An experimental NVFP4-KV profile reached HTTP readiness but corrupted semantics; it was rejected before benchmarking. The published numbers below use the coherent profile only.

What actually runs

01

Mixed checkpoint

Compressed-tensors FP8 dense groups plus NVFP4 expert weights.

02

Native target kernels

FlashInferCutlass NVFP4 GEMM and explicit FlashInfer B12X target MoE on sm_121a.

03

Correct MTP split

Two draft tokens; Triton handles the unquantized draft MoE while B12X stays on the quantized target.

Methodology & reproducibility

Concurrency: random 2,048-token input / exact 512-token output, concurrencies 1–32, three repetitions; first repetition dropped and the final two averaged. Telemetry: 2-second nvidia-smi and host-memory sampling with benchmark phase labels. Runtime: one GB10, text-only model path, 65,536 maximum sequence length, no Marlin or emulation fallback.
